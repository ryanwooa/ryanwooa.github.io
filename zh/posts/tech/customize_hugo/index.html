1. 外观类#

1.1 主题文件位置调整#

在安装好主题后，可以将下图圈出来的文件复制到站点对应的目录下，这样就可以自定义博客的样式，而不会影响主题的更新。

Hugo在加载时会优先读取站点目录下的文件，如果没有则读取theme目录下的文件。

    let activeElement;

        const width = document.body.scrollWidth;

        if (!element.getClientRects().length) {
            return 0;
        }
        let rect = element.getBoundingClientRect();
        let win = element.ownerDocument.defaultView;
        return rect.top + win.pageYOffset;

1.3 修改字体#

在static/fonts/目录下添加想要使用的字体文件

  1. 全局字体

在/assets/css/extended/blank.css中添加以下样式

body{
    font-family: 'circular-book', sans-serif;
}
  1. 代码字体

在/assets/css/common/post-single.css中找到.post-content code,添加想要的样式：

.post-content code {
    font-family: Consolas, Monaco, 'Andale Mono', 'Ubuntu Mono', monospace;
}

1.4 代码边框圆角#

  • 在/assets/css/common/post-single.css中添加以下样式：
.post-content .highlight pre {
    background-color: var(--theme) !important;
    margin: 0;
}
  • 块引用的圆角样式依然在该文件中，修改以下样式：
.post-content code {
    border-radius: 6px;
}

2. 功能类#

2.1 content目录结构调整#

content中每个文件夹内都要添加一个 _index.md文件，该文件里面可以用Front Matter 用来控制标题或其它的展示

.
└── content
    └── about
    |   └── _index.md  // <- https://example.com/about/
    ├── post
    |   ├── _index.md   // <- https://example.com/post/
    |   ├── firstpost.md   // <- https://example.com/post/firstpost/
    |   ├── happy
    |   |   └── ness.md  // <- https://example.com/post/happy/ness/
    |   └── secondpost.md  // <- https://example.com/post/secondpost/
    └── quote
        ├── first.md       // <- https://example.com/quote/first/
        └── second.md      // <- https://example.com/quote/second/
  • 如果你不在文件夹内添加_index.md文件，那么访问该文件夹的时候会直接显示该文件夹下的所有文件，而不会显示文件夹的标题。添加_index.md文件后，访问该文件夹时会显示_index.md文件中的内容。
  • 有时不添加_index.md文件会出现访问路径出错的情况，所以最好还是添加上。

更多参考：Hugo-Content organization

2.2 多语言模式#

  1. 配置hugo.yml的lauguages为多个语言
    languageCode: en-us
defaultContentLanguage: en 

languages:
en:
    languageName: "English"
    weight: 1  
    menu:
        main:
            - name: Articles
            url: posts/
            weight: 1
            - name: Tags
            url: tags/
            weight: 10
            - name: Search
            url: search/
            weight: 11

zh:
    languageName: "中文"
    weight: 2
    menu:
        main:
            - name: 文章
            url: posts/
            weight: 1
            - name: 标签
            url: tags/
            weight: 10  
            - name: 搜索
            url: search/
            weight: 11
  2. 翻译content中的内容, 有两种组织方式：

    1. 建立不同语言的文件夹，将对应内容放到对应的文件夹中，如下：

      .
└── content
    ├── en
    |   └── _index.md
    |   └── about.md
    ├── zh
    |   └── _index.md
    |   └── about.md

      我在尝试该方法时，可能是没有设置好文件夹的_index.md文件，所以导致无法显示任何内容，所以采用了下面的方法。

    2. 为每个文件设置多语言版本，例如：

      .
└── content
    └── about
    |   └── _index.md  // <- https://example.com/about/
    |   └── about.en.md
    |   └── about.zh.md

      在about.en.md文件中添加title字段，如下：

      title: "About"

      在about.zh.md文件中添加title字段，如下：

      title: "关于"

      这样就可以实现多语言切换了。

更多参考: Hugo-Multilingual Mode

2.3 搜索功能#

  1. 在content目录下新建search.md文件，内容如下：
    title: "Search" # in any language you want
layout: "search" # necessary for search
# description: "Description for Search"
summary: "search"
placeholder: "Please enter search content"
  2. 配置hugo.yml, 添加以下内容
    menu:
    main:
        -identifier: Search
        name: Search
        url: search
        weight: ...

随后就可以在nav中看到search选项了，点击即可搜索。

2.5 流量统计#

使用不蒜子流量统计 ，在/layouts/common/footer.css中对应位置添加如下代码：

{{- if not (.Param "hideFooter") }}
<footer class="footer">
    {{- if site.Copyright }}
		...
    {{- end }}
    <script async src="//busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"></script>
    <span id="busuanzi_container">
        <link rel="stylesheet" href="//maxcdn.bootstrapcdn.com/font-awesome/4.3.0/css/font-awesome.min.css">
        Visitors: <span id="busuanzi_value_site_uv"></span>
        Views: <span id="busuanzi_value_site_pv"></span>
    </span>
</footer>
{{- end }}

随后即可在footer中看到访问量和浏览量。

3. 优化类#

3.1 以新标签页打开链接#

在/layouts/_default/_markup/ 下新建render-link.html文件，添加如下内容：

<a href="{{ .Destination | safeURL }}"{{ with .Title}} title="{{ . }}"{{ end }}{{ if strings.HasPrefix .Destination "http" }} target="_blank" rel="noopener"{{ end }}>{{ .Text | safeHTML }}</a>

会自动为所有文章内的链接加上了 target="_blank" rel="noopener"
