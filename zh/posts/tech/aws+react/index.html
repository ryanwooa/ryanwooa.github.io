使用AWS部署React全栈项目

1. 背景#

近期和朋友们开发了一个全栈项目，包括React前端和Express后端，保存在一个github仓库中。现在想把这个项目上线，部署出来。

对于react项目来说有不少免费部署平台，比如vercel，digitalOcean。但是考虑这个项目中同时包含前端和后端，就考虑把他们部署在一个地方。参考了许多资料后决定把这个全栈项目部署到AWS云上。

本文主要介绍如何把一个完整的project部署的AWS EC2上。

2. 创建AWS EC2实例#

  1. 登录到AWS后，打开EC2
  2. 选择 Launch Instance
  3. 在镜像列表中选择Ubuntu server，选择free tier的即可
  4. Instance type选择t2.micro
  5. 设置key pair
  6. Network settings：因为要通过互联网进行访问，所以需要允许http和https访问
  7. 剩下的配置内容根据你的需要进行配置。
  8. 然后就可以单击Launch instance来启动服务器了。

3. 配置EC2实例环境#

  1. 连接到创建的EC2实例上（用户名填写root），连接后会在新网页打开ubuntu的终端

  2. 运行以下指令更新ubuntu server，完成后关闭这个网页，然后重新connect一次

    sudo apt-get update

  3. 运行以下指令安装必要的package

    sudo apt-get install curl          //安装curl
curl -sL https://deb.nodesource.com/setup_14.x | sudo -E bash -
sudo apt-get install -y nodejs     //安装nodejs
sudo npm install pm2@latest -g     //安装pm2
sudo apt-get install nginx -y      //安装nginx

//安装完成后可以使用一下命令查看安装版本
node -v
npm -v
nginx -v

  4. 转到EC2的安全，配置EC2的安全组策略 在inbound策略中添加你需要使用的端口号，我的项目中需要给后端开通3000和5050，就创建了两条

  5. 完成以上步骤之后，使用EC2的Public IPv4 DNS或 Public IPv4 address在新标签页中打开，就能看到nginx的欢迎页面模板

4. 下载React项目并永久启动#

因为我是已经做好的React项目，所以直接使用git来下载项目

  1. 下载Git项目

    git clone https://github.com/你的项目地址

//随后需要输入用户名和你自己的token进行身份验证
//token在个人的设置页面中，开发者选项中申请

  2. 跳转到react项目目录，然后build项目。需要确认编译后输出文件的名称，后面会用到。

    我的项目中输出文件夹是/dist ，

    cd /projectname/
npm install
npm run build

  3. 修改nginx配置文件并启动项目

    • 跳转到/etc/nginx路径下，编辑nginx.conf文件（这只是一种修改方式，更多方式请自行百度）

    • 修改user为root

    • 在http区域中添加一下代码，然后保存

      server{
  listen 80;
  server_name _;

  location /{
      root /root/project-name/frontend/dist;   //编译后端文件夹
      index index.html;                        //文件夹中需要包含index.html文件
      try_files $uri /index.html;
  }
}

    • 使用如下命令检查nginx配置，然后重启nginx

      sudo nginx -t  //检查
sudo systemctl restart nginx //重启服务
sudo tail -f /var/log/nginx/error.log   //如果出问题，使用这个查看错误日志

  4. 至此React项目就已经部署完成，可以刷新之前打开的public IP的页面查看是否能打开网页

5. 启动Nodejs并永久运行#

我的react部分和nodejs部分是在一个git仓库中，所以不在需要clone。如果在不同的git项目中，请先clone你的nodejs项目

  1. 安装运行包，测试是否能够正常运行

    npm install
npm start

  2. 用Ctrl+C停止运行

  3. 使用pm2永久运行

    cd src                      //转到node项目中app.js 或 index.js文件所在文件夹
npm2 start app.js           //使用pm2后台运行nodejs项目

//pm2常用指令
pm2 list                    //列出当前运行的进程
pm2 log                     //查看日志
pm2 delete all              //删除所有进程

  4. 至此，nodejs服务已经启动完成，可以在你的浏览器中输入你的API进行测试。

6. 一些注意事项#

  • 如果你的项目中有记录localhost地址的环境变量，记得修改localhost或127.0.0.1为当前EC2的public DNS
  • 如果有Google OAuth2.0，需要在其设置中添加EC2的public DNS来允许访问。
